# 系统启动说明

## 问题诊断

如果提单页面没有显示数据，可能的原因：

1. **后端服务器未启动**（最常见）
2. API 地址配置错误
3. 数据库未初始化

## 解决方案

### 方案 1：启动后端服务器（推荐）

1. **初始化数据库**（只需运行一次）：
```bash
cd server
npm install  # 如果还没安装依赖
npm run init-db
```

2. **启动后端服务器**：
```bash
cd server
npm run dev
```

服务器将在 http://localhost:3000 启动

3. **启动前端**（新开一个终端）：
```bash
npm run dev
```

### 方案 2：同时启动前后端

```bash
npm run dev:all
```

### 方案 3：仅使用前端（降级方案）

如果后端服务器未启动，系统会自动使用 mock 数据作为降级方案，数据会正常显示。

## 检查后端服务器状态

```bash
# 检查端口 3000 是否被占用
lsof -ti:3000

# 或者访问健康检查接口
curl http://localhost:3000/api/health
```

## 常见问题

### 1. 提单页面显示为空

**原因**：后端服务器未启动

**解决**：
- 启动后端服务器：`cd server && npm run dev`
- 或者系统会自动使用 mock 数据（已添加降级方案）

### 2. 数据库错误

**原因**：数据库未初始化

**解决**：
```bash
cd server
npm run init-db
```

### 3. 端口冲突

**原因**：3000 端口被占用

**解决**：
- 修改 `server/.env` 文件中的 `PORT` 配置
- 或关闭占用端口的程序

## 验证

1. 打开浏览器控制台，查看是否有错误信息
2. 如果看到 "⚠️ 后端服务器未启动，当前使用 mock 数据" 的提示，说明降级方案已生效
3. 如果看到 API 错误，检查后端服务器是否正常运行
